## Sequential two-shard collection and validation

The exhaustive collection was run sequentially in two shards because the raw
type and dependency tables are large. Both harnesses enumerate the same
deterministic 223-module environment and use the disjoint partition rule
`audited_environment_index_mod_2`: shard `i` emits a declaration exactly when
its audited-environment index is congruent to `i` modulo two.

[`run_recert_v4_shard.py`](scripts/run_recert_v4_shard.py) holds a global
exclusive V4-writer lock, refuses to overwrite a pre-existing output
directory, checks the frozen source manifest before and after Lean, and
requires every raw output to be nonempty. The two raw output directories are
`/private/tmp/hdp_v4_78da87_shard0` and
`/private/tmp/hdp_v4_78da87_shard1`.

During recovery from duplicate supervisor framing, each successful shard
transcript was passed through
[`normalize_recert_v4_shard_log.py`](scripts/normalize_recert_v4_shard_log.py).
The normalizer preserves the Lean diagnostic body, accepts only a final
successful exit, reconstructs the timestamped header from the authoritative
footer, and emits exactly one header and one `exit_code: 0` footer. It does not
alter any axiom, type, binder, dependency, calibration, or module table.

[`merge_pass07_v4_shards.py`](scripts/merge_pass07_v4_shards.py) then rejects
the merge unless:

- both normalized transcripts are closed successful runs of the expected
  harnesses;
- both shards report byte-identical complete module evidence;
- both private/known-Exercise calibration pairs pass;
- audit and type declaration identities are disjoint within and exhaustive
  across the two partitions;
- every binder key is unique and every declared telescope is complete;
- exact declaration, binder, and dependency totals hold; and
- the source manifest is still current.

The merge summary deliberately records the independent V7 declaration-universe
comparison as `DEFERRED_TO_V7_ENVIRONMENT_NAME_SET_RECONCILIATION`. This is not
a missing V4 partition: identical complete environment enumerations, the
mod-two partition, disjoint declaration identities, complete binder keys, and
the enforced exact totals close V4's own collection. V7 supplies an additional
independent name-set cross-check.

[`install_recert_v4_merge.py`](scripts/install_recert_v4_merge.py) validates
the merge summary, every artifact hash, and every exact row total before
atomically installing the canonical `recert_axiom_*` files. The merge must
name the exact current digest and current harness hash; no earlier-digest
fallback is accepted. The installer also preserves both normalized shard logs
and creates one aggregate transcript with one successful exit footer. Finally,
[`axiom_audit.py`](scripts/axiom_audit.py) validates schemas, module coverage,
types, telescope binders, direct dependencies, calibration, axiom sets, and
the exact V3 declaration allowlist.

## Limitations

`Lean.collectAxioms` establishes the transitive logical-axiom dependencies of
elaborated declarations; it does not establish that a statement is faithful
to the book, non-vacuous, or mathematically useful. V6 and V7 own those
semantic questions. Ordinary hypotheses leave no axiom trace, so V10
separately audits never-discharged conditional interfaces.

An environment audit cannot see a never-imported file by itself. V2 closes
that structural gap by proving there are no orphans, and V1 proves that all
declared surfaces build. V7's independent declaration-name-set comparison is
an additional cross-check rather than a prerequisite for the exhaustive
mod-two V4 merge.

The 228 `sorryAx` rows are known, marked Exercise debt, not completed proofs.
The Lean kernel and hash-pinned dependency packages remain the external trust
boundary; V4 does not re-audit every Mathlib or Batteries constant.
